# 资源缓存机制

## 概述

PyWebron 在 Rust 端实现了透明的资源缓存机制，无需 Python 端任何配置或代码。

## 工作原理

### 自动缓存

当使用 `content_dist` 参数加载本地资源时，所有通过自定义协议（`app://`）请求的资源会自动缓存到内存中：

```python
app.window.register_window(
    title="我的应用",
    content_dist=r"E:\works\myapp\dist",  # 自动启用缓存
)
```

### 缓存流程

1. **首次请求**：从磁盘读取文件 → 存入内存缓存 → 返回给浏览器
2. **后续请求**：直接从内存缓存返回（速度提升 10-100 倍）

### 性能对比

| 场景 | 首次加载 | 刷新页面 | 性能提升 |
|------|---------|---------|---------|
| 小文件 (< 100KB) | 1-5ms | 0.01-0.1ms | 10-50x |
| 大文件 (> 1MB) | 10-50ms | 0.1-1ms | 50-100x |

## 特性

### ✅ 完全透明
- 无需任何配置
- 无需编写缓存代码
- 自动管理缓存生命周期

### ✅ 智能日志
- **生产模式**：静默运行，无日志输出
- **开发模式**：仅记录耗时超过 5ms 的资源加载

### ✅ 内存安全
- 使用 `DashMap` 实现线程安全
- 缓存随应用退出自动释放
- 典型内存占用：2-50MB（取决于资源大小）

## 适用场景

### ✅ 推荐使用
- 本地 dist 目录（`content_dist`）
- 静态资源较多的应用
- 需要频繁刷新的开发场景

### ⚠️ 不适用
- 远程 URL（`content_url`）- 浏览器自带缓存
- 单个 HTML 文件（`content_path`）- 无需缓存

## 技术细节

### 缓存键
使用文件的完整路径作为缓存键：
```
E:\works\myapp\dist\assets\main.js
```

### MIME 类型支持
自动识别 30+ 种文件类型：
- 脚本：js, wasm
- 样式：css
- 图片：png, jpg, svg, webp, gif
- 字体：woff, woff2, ttf, otf, eot
- 视频：mp4, webm
- 音频：mp3, wav
- 其他：json, xml, pdf, txt

### 缓存策略
- **缓存时机**：首次请求时
- **缓存范围**：所有通过 `app://` 协议请求的资源
- **缓存有效期**：应用运行期间永久有效
- **缓存清理**：应用退出时自动清理

## 开发建议

### 开发模式
在开发时，如果修改了资源文件：
1. 重启应用以清空缓存
2. 或使用浏览器的硬刷新（Ctrl+Shift+R）

### 生产模式
- 缓存会自动工作，无需任何操作
- 用户刷新页面时体验更流畅
- 减少磁盘 I/O，降低系统负载

## 示例

### 基本使用
```python
from pywebron import App

app = App()

# 缓存自动启用，无需任何配置
app.window.register_window(
    title="我的应用",
    content_dist=r"./dist",
)

app.run()
```

### 多窗口共享缓存
```python
# 所有窗口共享同一个缓存
app.window.register_window(title="窗口1", content_dist=r"./dist")
app.window.register_window(title="窗口2", content_dist=r"./dist")
# 第二个窗口会直接使用第一个窗口加载的缓存
```

## 常见问题

### Q: 缓存会占用多少内存？
A: 取决于资源大小，典型应用 2-50MB。

### Q: 如何清空缓存？
A: 重启应用即可。缓存不会持久化到磁盘。

### Q: 缓存会影响启动速度吗？
A: 不会。缓存是按需加载的，不影响启动速度。

### Q: 可以禁用缓存吗？
A: 缓存是内置的优化，无法禁用。但它不会产生任何负面影响。

```

```

在生产模式下（`cargo build --release`），日志会被自动禁用以提升性能。
